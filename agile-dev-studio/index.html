Student Project Brief

The Challenge

Students are tasked with solving a real digital problem for a community partner, such as a school club portal or a small business inventory tool.

Working in teams, students move from an initial concept to a functional MVP, following a structured and teacher-guided development process.

Final Deliverables

  • A functional web application with backend logic and a user interface
  • A professional GitHub README with setup instructions and technical explanation
  • A project Kanban board showing planning, task ownership, and progress
  • An individual post-mortem reflection analyzing challenges and growth

Project Phases (Sprint Cycle)

  • Discovery & Design

    Students analyze user needs, define user stories, and create wireframes and data maps before writing code.

  • Development Sprints

    Teams build the application in short, guided sprints with regular checkpoints and peer feedback.

  • Quality Assurance (QA)

    Students test each other's work to identify bugs, improve usability, and practice constructive technical feedback.

  • Demo Day

    Teams present their final product to classmates and stakeholders, explaining both the solution and the development process.

Roles & Differentiation

This project uses structured role-based collaboration to support mixed-ability classrooms.

Project Manager

  • Oversees timeline and task organization
  • Maintains documentation and Kanban board
  • Acts as client liaison

Backend Lead

  • Implements data logic and server-side functionality
  • Manages validation, security considerations, and core features

UX/UI Lead

  • Designs user interface and improves usability
  • Applies accessibility principles aligned with WCAG standards

Technical Writer

  • Translates technical decisions into clear documentation
  • Produces user-friendly setup and usage guides

Roles can rotate or be combined based on class size and student readiness.

Skills Developed

Technical Skills

  • Full-stack fundamentals (input, storage, output)
  • Guided version control using Git
  • System design and data modeling
  • AI-assisted debugging and documentation (See AI Use Policy below)

Professional & Career Skills

  • Agile methodology through simplified daily stand-ups
  • Task management and accountability using Kanban boards
  • Technical communication and documentation
  • Collaboration, feedback, and professional reflection

Assessment Approach (Process-Based)

Assessment prioritizes both the final product and the development process.

Criteria Weight
Product Reliability 40%
Professional Workflow (Kanban + Git) 30%
Documentation & Reflection 20%
Collaboration & Participation 10%

Teacher Notes

  • Basic programming concepts (variables, loops, conditionals)
  • Introductory HTML/CSS
  • Teacher-provided project templates and starter code
  • Step-by-step modeling of core workflows (Git commits, Kanban updates, code structure)
  • Regular checkpoints to prevent cognitive overload (design review, sprint demo, QA test)
  • Scaffolded Git usage and code structure with provided templates
  • Weekly sprint planning with clear deliverables and time boxes
  • Teacher-provided: starter template, example README structure, Kanban board template, and weekly checklists

This project is designed to be fully teacher-guided while allowing student ownership and creativity.

Allowed

  • Using AI tools for debugging, syntax help, and documentation research
  • Using AI to draft small snippets or starting points, then revising and explaining the final solution
  • Exploring alternative solutions and learning new concepts

Not Allowed

  • Copy-pasting code without understanding or explanation
  • Using AI to complete entire features without student contribution
  • Submitting AI-generated work as original without modification

Required Documentation

  • Students must note the AI tool used and what it supported (debugging, explanation, documentation)
  • Students must be able to explain any AI-assisted code line-by-line
  • Post-mortem reflection must include how AI tools were used and what was learned

Middle School Adaptation (Grades 7–8)

This project can be adapted for middle school by:

  • Reducing project scope (single-page apps, simplified features)
  • Providing pre-built templates with clear TODOs
  • Using checklist-based workflows instead of open-ended sprints
  • Increasing teacher modeling and structured support at each phase
